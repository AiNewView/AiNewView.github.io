Gemini 3顶级玩家才知道的深度用法！

发表于 2026-01-11

一、 通过系统指令(System Instructions)建立“长期记忆”

1.1 系统指令的定义

在 Gemini 3 系统中，系统指令（System Instructions） 被定义为一种**“前置元指令”**。它与普通对话（User Prompt）分开，在模型接触到你的具体问题之前，先从底层定义它的“出厂设置”。

1.2 系统指令的特点

系统指令是一组预设的规则、角色和背景信息。它的特点是：

  • 全局性：它作用于整个对话生命周期，无论对话进行了多少轮，AI 都会始终优先遵循这些指令。

  • 隔离性：在技术实现上，它与用户输入的具体任务内容分离。这能有效防止 AI 在处理大量数据时，因为任务太复杂而“忘记”最初的要求。

  • 高权重：Gemini 3 会将系统指令视为最高优先级的行为准则。

1.3 系统指令能做什么？

通过设置系统指令，你可以从以下三个维度彻底改变 Gemini 的表现：

  • 定义人格（Persona）：

    • 示例：“你是一位了解AI发展前世今生的专家，语言简练，代码效率高。”

  • 定义客户（user）：

    • 示例：“我是一位从事AI算法8年的专业人员，目前居住在北京，关注AI的各类最新进展，包括LLM、搜索、Agent、具身智能等。”

  • 定义输出规范（Output Constraints）：

    • 示例：“始终以 markdown 格式输出，不要包含任何多余的解释文字。”

  • 定义知识范围（Context Grounding）：

    • 示例：“你只允许根据我提供的文档回答问题。如果文档中没有相关信息，请回答‘未在源文件中找到’，不得自行联网搜索。”

1.4 系统指令如何作用

  • Google AI Studio (开发者端)：在左侧面板有专门的 System Instructions 输入框。
  • Gemini 网页端 (Gems)：通过创建“Gem”功能，你填写的“自定义指令”本质上就是系统指令。
  • API 调用：在 GenerativeModel 初始化时通过 system_instruction 参数传入。

二、 风格统一

要让大语言模型（LLM）生成的内容贴近你的个人风格，核心在于**“样本提供”和“规则定义”**。你可以把这个过程想象成在给一位代笔作家做深度培训。

2.1 提供“少样本”示例 (Few-Shot Prompting)

不要只告诉 LLM“写得像我”，而是直接把你的文字喂给它。

  • 直接指令
    • 指令模板：“以下是我写的几段文字，请分析我的语言风格（包括语气、常用词汇、句式长度、幽默感等）。分析完后，请模仿这种风格帮我写一段关于[新主题]的内容。”
    • 示例：[粘贴你的朋友圈、邮件或工作报告原文]
    • 一个高阶的 Prompt 技巧示例：“你现在是我的私人文案助理。我会给你三段我以前写的文字。请学习我的节奏感（比如长短句的交替）和用词偏好（比如我喜欢用具体的例子而不是宏大的概念）。
      • 我的文字： [样本1] [样本2] [样本3]
      • 任务：请用刚才学到的风格，写一封回复客户拒绝合作的邮件，要表现得‘虽然遗憾但依然专业，且不失气场’。”
  • NotebookLM模板：以工作总结为例，将过往的工作总结上传给NotebookLM，结合“指令模板”，让Gemini学习用户的风格。

2.2 使用“系统指令”或“自定义指令” (Custom Instructions) —— 最持久

如果你希望模型永远记住你的风格，而不是每次聊天都重新训练，你应该使用 Gemini 的 “Gems” 或 ChatGPT 的 “Custom Instructions”。

  • 操作建议： 在设置中输入：“在所有回复中，请遵循以下风格：1. 保持技术性但用大白话解释；2. 回答要直接，不要客套；3. 使用中英混学风格（如果你平时说话就这样）。”
